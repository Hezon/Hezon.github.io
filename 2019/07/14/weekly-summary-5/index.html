浏览器常驻线程分析

发表于 2019-07-14 | 分类于 技术周报

在浏览器内核里，有一些常驻线程，如下

  • GUI 渲染线程
  • JS 引擎线程
  • 定时器触发线程
  • 事件触发线程
  • http 异步请求线程

GUI 渲染线程

当页面需要重绘时，该线程就会被执行。

JS 引擎线程

JS 引擎线程负责解析运行 javascript 脚本，在执行代码的过程中若调用一些浏览器 api，就把这些调用浏览器 api 的任务分发给其他相关的线程去运行。运行结束后，这些任务回调函数就会进入任务队列等待。在 js 执行栈为空后，这些任务就通过事件循环进入执行栈运行。

定时器触发线程

当 js 主线程执行栈执行了 setTimeout 或 setInterval 时，就会把这个定时任务交给定时器触发线程去执行。计时结束后，把回调函数添加到任务队列里。

事件触发线程

事件触发线程用来监听一些浏览器的操作，比如鼠标事件、键盘事件。当事件触发后，就把回调函数放入任务队列中。

HTTP 异步请求线程

该线程负责 http 请求，在得到相应时，把回调函数放入任务队列中，等待 js 引擎获取执行。
